LumoSQL Releases and Downloads

NOTE - This is a dummy page at the moment. None of the links work. If you have feedback we'd love to hear it. LumoSQL is tricky to explain simply, and this dummy page is to experiment with how a public release of LumoSQL should be presented. We assume that if you want to download LumoSQL you know it is a drop-in replacement for the SQLite library (eg /usr/lib/libsqlite3.so depending on your operating system) and that you are either writing an application that uses the sqlite3 library, or, you are using the commandline version (eg /usr/bin/sqlite3). We would also hope and expect LumoSQL to be packaged in operating systems, so that means the packagers have to understand how we do LumoSQL releases.

LumoSQL is available in several ways, depending on your needs. We will not be providing binaries until LumoSQL v1.0 is released, so your choices are to get all the source for generating 700+ versions and combinations, or, to select just the sources that LumoSQL tools generate relevant to your needs. None are large downloads, and the Not-forking tool does cache downloads and reconstruct trees from local versions as much as possible.

If you have problems getting a LumoSQL release you can contact us and we will do our best.

LumoSQL Trunk

'Trunk' means the same as 'master', 'default' or 'HEAD'. It's the current latest development version.

If you are a developer curious about which of the many combinations might be right for your application, then you want the ability to generate everything the LumoSQL tools can give you. This includes any version of modified SQLite with encryption, privacy, backend, benchmarking etc features. For this you will need the Fossil trunk SCM tree, or its Git mirror, or the Trunk source tarball:

LumoSQL trunk

fossil clone https://lumosql/src/lumosql
cd lumosql
fossil update # for current trunk
fossil update version-0.7 # for a specific LumoSQL version
fossil tag list | grep version # to list all versions
GitHub mirror of LumoSQL trunk git clone https://github.com/LumoSQL/lumosql
cd lumosql
git checkout # for current trunk
git checkout version-0.7 # for a specific LumoSQL version
git tag | grep version # to list all versions
Tarball of trunk

wget https://lumosql.org/src/lumosql/tarball/trunk/lumosql.tar.gz

wget https://lumosql.org/src/lumosql/tarball/version-v0.7/lumosql.tar.gz   #for a specific LumoSQL version

 

Downloads for Specific use cases

You might just be interested in one aspect of LumoSQL. There are still many combinations of trees, but still there will be far fewer options. All of the following are the trees as generated by the current trunk of LumoSQL, rather than any particular release.

 

LumoSQL generated product Download versions
LumoSQL Flavour 1 - SQLite modifications, native BTree. This is SQLite modified to have hidden columns and other metadata to support per-row checksums, per-row encryption and per-table encryption. An ordinary unmodified SQLite binary can open and operate on databases created by binaries compiled in these trees. However only the code in these trees has the additional commands required to see and manipulate this metadata.
LumoSQL Flavour 2 - SQLite modifications, LMDB backend. This is SQLite with hidden columns and other metadata just as in Flavour 1. The key-value store is also LMDB, so unmodified SQLite binaries cannot open databases these trees create. The new SQLite commands work exactly as for Flavour 1. LMDB inspection tools can however see databases created by binaries made from these trees.
LumoSQL Flavour 3 - Unmodified SQLite, LMDB. This is ordinary SQLite in every way, except the key-value store is LMDB. These trees help you answer the question "what if I used a 21st-century memory-mapped database model instead of a 20th-centurl Write-ahead log?" The answer isn't necessarily obvious, but it is a very good question you can now ask with minimal effort.
